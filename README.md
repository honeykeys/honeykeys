# Karl Nuyda

Notes to myself about machines that almost think.

I love working with models. They know too much and remember too little. They make things up. They're overconfident and sycophantic. They remind me of someone.

The work is not more output. It's only what holds up. A model can't judge its own work, so nothing I build trusts itself. That's the whole design.

I came to code through English Literature. A program is prose that runs. Hard in the structure, soft in the hue.

### Things I'm making

- **resume-loop**: honesty as an engineering problem.
- **tingin**: the handoff between nurses, taught to a machine.
- **hargow**: a classroom that pays attention back.
- **kyb-pdf-parser**: meaning read out of bank statements.
- **ferrokit**: the starter kit I reach for.
